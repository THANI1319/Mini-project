🔬 Methodology

How It Works

  • You write Python code in the editor.
  • The backend runs the same code in two isolated Docker containers:
    • Container simulation: lightweight python:3.9-slim image
    • VM simulation: heavier python:3.9 image (simulates VM overhead)
  • While executing, we measure startup time, CPU usage, and memory usage using Docker stats.

Metrics

  • Startup time: time until code begins execution (seconds).
  • CPU usage: average % CPU during execution.
  • Memory usage: peak resident memory (MB).

Security

  • Each run is isolated in a fresh container.
  • Network disabled, memory/cpu limited, and timeout (30s).
  • No persistent storage – containers are removed after execution.
